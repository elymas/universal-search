   Plaintext secrets were rendered into a chart-managed Secret. DO NOT use this

   Secret yourself:
     kubectl create secret generic <name> \
       --from-literal=POSTGRES_PASSWORD=... --from-literal=MEILI_MASTER_KEY=... \
       --from-literal=LITELLM_MASTER_KEY=... --from-literal=SEARXNG_SECRET=... \
       --from-literal=OIDC_CLIENT_SECRET=... --from-literal=JWT_SIGNING_KEY=... \
       --from-literal=SESSION_SECRET=...
   then set secrets.existingSecret.name=<name>.
{{- end }}

   kubectl --namespace {{ .Release.Namespace }} port-forward \
     svc/{{ include "usearch.componentName" (dict "root" . "component" "api") }} \
     {{ .Values.usearch.api.service.port }}:{{ .Values.usearch.api.service.port }}

   Then: curl http://localhost:{{ .Values.usearch.api.service.port }}/healthz

2. Schema migration ran as a pre-install/pre-upgrade hook Job
   ({{ include "usearch.fullname" . }}-migrate) BEFORE the app pods started.

   helm test {{ .Release.Name }} --namespace {{ .Release.Namespace }}
