# Roadmap — Universal Search Engine V1

## 1. Milestone Map

```
M1  Foundation            →  M2  Single-source          →  M3  Fanout + Index
(bootstrap, infra)           (one adapter, e2e thin)       (full adapter suite)

M4  Synthesis (basic)     →  M5  /deep multi-agent      →  M6  Team plane
(gpt-researcher wrap)        (STORM + 4-agent)             (auth, shared index)

M7  Surfaces              →  M8  Observability +         →  M9  V1 release
(CLI, MCP, Skill, UI)        Evaluation (DeepEval)         (docs, helm, tag)
```

Milestones are sequenced by dependency. M1 and early M2 can parallelize if two developers are available. Team mode (`--team` flag enabled) parallelizes SPECs within a milestone.

## 3. Parallelization Plan (Team Mode)

With `--team` enabled (Agent Teams), within a milestone, independent SPECs execute in parallel:

| Milestone | Parallelizable SPECs |
|-----------|----------------------|
| M1 | SPEC-BOOT-001 + SPEC-OBS-001 + SPEC-LLM-001 (3-way) |
| M2 | SPEC-ADP-001 + SPEC-ADP-002 + SPEC-CLI-001 (3-way, after SPEC-IR-001) |
| M3 | All SPEC-ADP-* (7-way), SPEC-IDX-* (3-way) — gated on SPEC-FAN-001 |
| M4 | SPEC-SYN-002 + SPEC-SYN-003 + SPEC-SYN-004 (3-way) |
| M5 | SPEC-DEEP-001 + SPEC-DEEP-002 (parallel), then SPEC-DEEP-003 + SPEC-DEEP-004 |
| M6 | SPEC-AUTH-* (3-way) + SPEC-IDX-004 (parallel with auth) |
| M7 | SPEC-MCP-001 + SPEC-CLI-002 + SPEC-UI-001 (3-way), then SPEC-SKILL-001 + SPEC-UI-002 |
| M8 | SPEC-EVAL-* (3-way) + SPEC-SEC-001 |

Cross-milestone gates:

- M2 must complete before M3 (fanout needs a working adapter pattern).
- M3 must complete before M4 (synthesis needs results).
- M6 can start in parallel with M5 after M4 completes (auth does not depend on /deep).
- M7 depends on M3 (surfaces need working query path).
- M8 is continuous from M3 onward (benchmarks track regressions milestone-to-milestone).

## 4. Priority Labels (per MoAI time-estimation rule)

- **Priority High**: M1–M3 (foundational, blocking everything).
- **Priority Medium**: M4–M6 (value-delivering).
- **Priority Low**: M7 (surfaces beyond CLI), M8 (polish), M9 (release).

## 5. Milestone Exit Criteria (summary — full criteria in each SPEC)

| Milestone | Exit Criteria |
|-----------|---------------|
| M1 | `docker-compose up` starts all deps; `usearch --version` runs; CI green. |
| M2 | `usearch query "hello world"` returns Reddit + HN results with one synthesized paragraph + citations. |
| M3 | All 12+ adapters pass contract tests; `usearch query` returns fused results across ≥5 adapters; Korean query returns Naver results ranked first. |
| M4 | Citation faithfulness ≥0.85 on golden set; streaming responses under 8s p50. |
| M5 | `usearch deep "..."` returns STORM-style report with ≥10 cited sources in ≤5min. |
| M6 | Team with 5 users: per-user quotas enforced, shared index dedup hits ≥30%, audit log complete. |
| M7 | Claude Skill installs from marketplace; MCP server connects from Claude Code + Codex + Gemini CLI. |
| M8 | DeepEval CI gate at ≥0.85; adapter success rate dashboard live; security pass clean. |
| M9 | `v1.0.0` tagged; Helm chart deployable; docs site live. |

## 6. Post-V1 Backlog (explicit deferral list)

These are deliberately **not** in V1. They are recorded here so scope creep has a known destination:

- Personal context adapters (Gmail / Calendar / Drive / Obsidian / Slack) — requires OAuth + privacy framework
- SaaS / hosted multi-tenant offering — requires licensing review (SearXNG AGPL)
- Image / video generation
- Agentic task execution (beyond retrieval)
- Finetuned local LLM for synthesis
- iOS / Android clients
- Federated search across multiple Universal Search deployments
- Real-time collaborative research sessions (multi-user on one query)

## 7. How to pick up work

1. `/moai plan SPEC-XXX-NNN` — expand a backlog SPEC into a full EARS document under `.moai/specs/`.
2. `/moai run SPEC-XXX-NNN` — implement via DDD/TDD per `quality.yaml`.
3. `/moai sync SPEC-XXX-NNN` — doc sync + PR.

Always start with M1. SPEC-BOOT-001 is the entry point.
