# 문제 해결

컴포넌트별로 정리된 일반적인 문제와 해결책입니다.

## 설치 및 빌드

### `make build` 실패 — Go 버전 불일치

| 항목 | 내용 |
|---|---|
| **증상** | `go: go.mod requires Go >= 1.25` |
| **원인** | Go 버전이 1.25 미만 |
| **해결** | Go 업그레이드: `go install golang.org/dl/go1.25.0@latest && go1.25.0 download` |
| **확인** | `go version`이 1.25+ 표시 |

---

### `make compose-up` 실패 — 포트 충돌

| 항목 | 내용 |
|---|---|
| **증상** | `Error: bind: address already in use` (포트 6333, 7700 등) |
| **원인** | 다른 서비스가 해당 포트 사용 중 |
| **해결** | 충돌하는 서비스를 중지하거나 `.env`에서 포트 재정의 (예: `QDRANT_PORT=6334`) |
| **확인** | `docker compose ps`에서 모든 서비스가 healthy |

---

## 검색 및 쿼리

### `usearch query`에서 결과 없음

| 항목 | 내용 |
|---|---|
| **증상** | `No results found` 또는 빈 출력 |
| **원인** | SearXNG 연결 불가, 모든 어댑터 비활성화, 또는 네트워크 차단 |
| **해결** | 1. `curl http://localhost:8080/search?q=test` 확인 (SearXNG). 2. `usearch sources list` 확인. 3. 방화벽/프록시 설정 확인 |
| **확인** | `usearch sources test searxng --query "test"` 성공 |

---

### 한국어 쿼리에서 한국어 결과 없음

| 항목 | 내용 |
|---|---|
| **증상** | `usearch query --locale ko "..."` 실행 시 영어 결과만 반환 |
| **원인** | 네이버 어댑터 비활성화 또는 자격 증명 누락 |
| **SPEC** | SPEC-CLI-002 |
| **해결** | `NAVER_CLIENT_ID` + `NAVER_CLIENT_SECRET` 설정; `usearch sources enable naver` 실행 |
| **확인** | `usearch sources list`에서 naver가 `enabled`; `usearch sources test naver --query "test"` 성공 |

---

### 합성 품질 저하 / 환각

| 항목 | 내용 |
|---|---|
| **증상** | LLM 답변이 소스와 일치하지 않음 |
| **원인** | LLM 모델의 저품질 합성; 불충분한 인용 |
| **해결** | `--timeout 60s`로 검색 시간 증가; 더 높은 품질을 위해 `usearch deep` 사용; `ANTHROPIC_API_KEY` 설정 확인 |
| **확인** | 출력에 비영점 신뢰도 점수와 함께 인용 표시 |

---

## MCP 통합

### Claude for Desktop에서 "usearch MCP 서버를 찾을 수 없음"

| 항목 | 내용 |
|---|---|
| **증상** | Claude에 MCP 서버가 표시되지 않음 |
| **원인** | `usearch-mcp` 바이너리가 PATH에 없거나 config.json 문법 오류 |
| **SPEC** | SPEC-MCP-001 |
| **해결** | 1. `which usearch-mcp` 확인. 2. `jq . config.json`으로 config.json 유효성 검사. 3. Claude for Desktop 재시작 |
| **확인** | Claude 재로드 — usearch가 MCP 서버 목록에 표시됨 |

---

### MCP 인증 실패

| 항목 | 내용 |
|---|---|
| **증상** | MCP 응답에서 `AUTH_ERROR` |
| **원인** | 토큰 만료 또는 무효 |
| **해결** | 새 토큰 생성: `usearch admin token create --name "claude-desktop" --expires 90d` |
| **확인** | `usearch login --status`가 유효한 세션 표시 |

---

## 데이터베이스 및 스토리지

### Qdrant 연결 거부

| 항목 | 내용 |
|---|---|
| **증상** | 포트 6333에서 `connection refused` |
| **원인** | Qdrant 컨테이너 미시작 또는 비정상 |
| **해결** | `docker compose restart qdrant`; `docker compose logs qdrant` 확인 |
| **확인** | `curl http://localhost:6333/health`가 `{"status":"ok"}` 반환 |

---

## 인증

### `usearch login`이 브라우저 단계에서 멈춤

| 항목 | 내용 |
|---|---|
| **증상** | 브라우저가 열리지 않거나 OAuth 콜백 실패 |
| **원인** | 잘못된 `OAUTH_REDIRECT_URL` 또는 콜백 포트를 차단하는 방화벽 |
| **SPEC** | SPEC-AUTH-002 |
| **해결** | `OAUTH_REDIRECT_URL=http://localhost:8081/auth/callback`이 OAuth 제공자 설정과 일치하는지 확인 |
| **확인** | 개발 환경에서 테스트 로그인 플로우 완료 |

---

## 성능

### `usearch deep`이 타임아웃됨

| 항목 | 내용 |
|---|---|
| **증상** | 심층 연구가 완료 전에 종료됨 |
| **원인** | 기본 5분 타임아웃 초과; 또는 LLM API 속도 제한 |
| **해결** | 타임아웃 증가: `usearch deep --timeout 15m "..."`; LLM API 할당량 확인 |
| **확인** | 더 짧은 쿼리가 타임아웃 내에 완료됨 |
