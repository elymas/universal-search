# 주요 사용자

Universal Search는 심층 연구가 필요하고, 여러 소스에 걸쳐 근거 있는 인용된 답변을 원하는 팀과 개인을 위해 만들어졌습니다.

## 주요 페르소나

### 연구 중심 엔지니어

**필요**: GitHub, arXiv, 문서, HN에 걸친 기술 심층 연구

**성공 지표**: 이전에 ≥15분 걸리던 것을 ≤60초에 근거 있는 답변 획득

**주요 워크플로**:
- `usearch query "Raft 합의 알고리즘에서 리더 선출은 어떻게 처리하나?"` — arXiv, GitHub, HN 교차 참조
- `usearch deep "이벤트 스트리밍에서 Kafka vs Pulsar 비교"` — 인용 포함 멀티 에이전트 심층 연구
- 코딩 세션 중 인라인 연구를 위한 Claude MCP 통합

---

### 제품/전략 리더

**필요**: 시장 + 소셜 + 뉴스 신호, 한국어 + 글로벌

**성공 지표**: 인용 출처와 함께 10분 이내 주간 보고서 작성

**주요 워크플로**:
- `usearch query "경쟁사 제품 X 시장 반응"` — 웹 + 소셜 + 뉴스 합성
- 비동기 공유를 위한 인용 마크다운 보고서 내보내기
- 팀 공유 쿼리 이력을 위한 웹 UI 대시보드

---

### 한국어 분석가 / 기자

**필요**: 네이버 뉴스, 다음, 한국어 RSS, 글로벌 소스와의 교차 참조

**성공 지표**: 한국어 쿼리가 번역된 요약 대신 한국어 소스를 우선 반환

**주요 워크플로**:
- `usearch query --locale ko "네이버 클로바 vs GPT-4o 비교"` — 한국어 로케일 검색
- 네이버 뉴스 + 다음 + 한국어 기술 블로그가 우선 순위
- 정확한 한국어 NLP를 위한 mecab-ko 토크나이저

---

### 팀 리더 (공유 메모리)

**필요**: 팀의 이전 쿼리가 재검색 가능한 자산이 됨

**성공 지표**: "누군가 이미 X를 연구했나?" 를 공유 인덱스에서 답변 가능

**주요 워크플로**:
- 팀 전체에 걸친 공유 Qdrant + Meilisearch 인덱스
- `usearch history`를 통한 쿼리 이력
- 다중 사용자 접근 제어를 위한 RBAC

## 적합하지 않은 사용 사례

- 코딩 보조 (GitHub Copilot, Cursor 사용)
- 작업 실행 (예약, 이메일 발송)
