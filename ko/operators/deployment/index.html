운영자 가이드배포

배포

Universal Search는 Docker Compose 스택으로 배포됩니다. 이 가이드는 조직 규모의 자체 호스팅 프로덕션 환경 배포를 다룹니다.

배포 옵션

옵션 1: Docker Compose (팀에 권장)

클론 및 구성

git clone <repo-url> && cd universal-search
cp .env.example .env.production
# 필수 값 입력 — 구성 참조 확인

스택 시작

docker compose -f deploy/docker-compose.yml --env-file .env.production up -d

상태 확인

docker compose ps           # 모든 서비스 정상
curl http://localhost:8081/health

옵션 2: Kubernetes (Helm — 계획 중)

Helm 차트 배포는 SPEC-DEPLOY-001에서 계획 중입니다. 해당 SPEC이 제공될 때까지 Docker Compose를 사용하세요.

데이터 영속성

상태 유지 서비스를 위한 영속 볼륨 구성:

volumes:
  qdrant_data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /data/qdrant
  meilisearch_data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /data/meilisearch
  postgres_data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: /data/postgres

영속 볼륨 없이는 컨테이너 재시작 시 모든 데이터(쿼리 이력, 인덱싱된 문서, 사용자 계정)가 사라집니다.

백업

정기적으로 백업하세요:

# Qdrant 스냅샷
curl -X POST http://localhost:6333/collections/{name}/snapshots
 
# Meilisearch 덤프
curl -X POST http://localhost:7700/dumps
 
# PostgreSQL
pg_dump -h localhost -U usearch usearch_db > backup.sql

관련 항목

  • 구성 레퍼런스 — 모든 환경 변수 (영어)
  • 인증 설정 — OAuth/OIDC 구성
  • SPEC-DEPLOY-001: Kubernetes/Helm 배포 (향후)
