# Naver 어댑터

<StatusBadge adapter="naver" />

Naver 어댑터는 Naver의 다목적 검색 API(블로그, 뉴스, 웹, 쇼핑, DataLab)를 쿼리합니다. **인증 필요** — Naver 개발자 애플리케이션 클라이언트 ID와 시크릿이 필요합니다.

## Overview

Naver는 대한민국의 주요 검색 엔진입니다. Universal Search Naver 어댑터는 블로그, 뉴스, 웹, 쇼핑, DataLab 트렌드 등 다양한 수직 검색을 지원합니다. 쿼리는 Naver OpenAPI 엔드포인트(`openapi.naver.com`)에 UTF-8로 전달됩니다.

**활용 사례:** 한국어 콘텐츠 검색, 한국 뉴스 집계, 한국 블로그 검색, DataLab 트렌드 데이터.

## Setup

**인증 필요.** 환경 변수로 Naver API 자격증명을 제공하세요.

**1단계 — Naver 개발자 애플리케이션 등록:**
1. [developers.naver.com](https://developers.naver.com/main/)에 접속
2. **내 애플리케이션** 에서 새 애플리케이션 생성
3. **검색 API** 서비스 활성화
4. **중요:** 애플리케이션 설정에서 Service URL 등록 (redirect allowlist에 `openapi.naver.com` 필요)
5. **Client ID**와 **Client Secret** 복사

**2단계 — 환경 변수 설정:**

```bash
export NAVER_CLIENT_ID=<NAVER_CLIENT_ID를_입력하세요>
export NAVER_CLIENT_SECRET=<NAVER_CLIENT_SECRET을_입력하세요>
```

**3단계 — 설정 확인:**

```bash
usearch query --source naver "인공지능 최신 논문"
# 기대값: 한국어 블로그/뉴스 결과, 401 오류 없음
```

**한국어 로케일 사전 요구사항:**
- 한국어 쿼리는 UTF-8로 Naver API에 그대로 전달됩니다 (어댑터 내 토크나이징 없음)
- 인덱스 사이드 한국어 토크나이징(mecab-ko)은 인덱싱 레이어(SPEC-IDX-003)가 담당합니다
- DataLab 엔드포인트는 검색 엔드포인트와 별도 rate 예산을 사용합니다
- [한국어 로케일 설정](/ko/operators/korean-locale-setup) 참고

## Query syntax

| Universal Search 필드 | Naver API 파라미터 | 비고 |
|-----------------------|-------------------|------|
| `Query.Text` | `query={text}` | UTF-8 한국어 텍스트 그대로 전달 |
| `Query.Filters["naver_vertical"]` | 엔드포인트 경로 결정 | `blog` (기본), `news`, `web`, `shop`, `datalab` |
| `Query.MaxResults` | `display={n}` | 최대 100; 기본값 25 |

## Rate limits

- **광고 제한:** 10 요청/분
- **적용 메커니즘:** HTTP 429 응답 처리 — 할당량 초과 시 Naver가 `Retry-After` 헤더와 함께 429 반환
- **업스트림 할당량 문서:** [Naver Developers API 할당량](https://developers.naver.com/docs/common/openapiguide/)
- **소진 동작:** 어댑터가 429 응답에서 `Retry-After`를 읽어 `CategoryRateLimited`를 반환합니다

## Error reference

공유 `Category` 참조는 [오류 분류 체계](/ko/reference/adapters/errors)를 확인하세요.

**Naver 특정 상태 코드 대응표 (`naver/client.go:87-110`):**

| HTTP 상태 | Category | 원인 | 운영자 조치 |
|-----------|----------|------|------------|
| 401 | CategoryPermanent | 잘못된 클라이언트 ID | `NAVER_CLIENT_ID` 환경 변수 확인 |
| 403 | CategoryPermanent | 접근 거부 / SSRF 가드 | Naver 개발자 콘솔에서 Service URL 등록 여부 확인 |
| 429 | CategoryRateLimited | 요청 할당량 초과 | `Retry-After` 대기; 요청 빈도 줄이기 |
| 4xx (기타) | CategoryPermanent | 잘못된 요청 | 쿼리 형식 및 수직 이름 확인 |
| 5xx | CategoryUnavailable | Naver API 사용 불가 | 잠시 후 재시도 |

## Troubleshooting

| 증상 | 원인 가능성 | 진단 명령 | 해결 방법 | 관련 SPEC |
|------|-----------|----------|----------|----------|
| 401 "Invalid client id" | 잘못된 `NAVER_CLIENT_ID` | 환경 변수 확인 | Naver 개발자 콘솔에서 올바른 클라이언트 ID 내보내기 | SPEC-ADP-008 |
| 유효한 쿼리에도 결과 0건 | Naver 콘솔에 Service URL 미등록 | Naver 애플리케이션 설정 확인 | `openapi.naver.com`을 허용된 Service URL로 등록 | SPEC-ADP-008 |
| SSRF 가드 차단 (403) | 허용목록에 없는 도메인으로 요청 | 어댑터 로그 확인 | 어댑터는 `openapi.naver.com`만 허용 (SPEC-SEC-001 D3) | SPEC-SEC-001 |
| 한국어 토크나이징 문제 | 인덱스 사이드에서 mecab-ko 미설정 | Meili 플러그인 상태 확인 | [한국어 로케일 설정](/ko/operators/korean-locale-setup) 따라하기 | SPEC-IDX-003 |
