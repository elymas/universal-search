Troubleshooting

Troubleshooting

Common issues and solutions organized by component.

Installation & Build

make build fails — Go version mismatch

FieldValue
Symptomgo: go.mod requires Go >= 1.25
CauseGo version below 1.25
SPECSPEC-CLI-001
FixUpgrade Go: go install golang.org/dl/go1.25.0@latest && go1.25.0 download
Verifygo version shows 1.25+

make compose-up fails — port conflict

FieldValue
SymptomError: bind: address already in use for port 6333, 7700, etc.
CauseAnother service is using the port
FixStop the conflicting service, or override the port in .env (e.g., QDRANT_PORT=6334)
Verifydocker compose ps shows all services healthy

Search & Queries

usearch query returns no results

FieldValue
SymptomNo results found or empty output
CauseSearXNG unreachable, all adapters disabled, or network blocked
Fix1. Check curl http://localhost:8080/search?q=test (SearXNG). 2. Check usearch sources list. 3. Check firewall/proxy settings
Verifyusearch sources test searxng --query "test" succeeds

Synthesis quality is poor / hallucinations

FieldValue
SymptomLLM answer doesn’t match the sources
CauseLLM model producing low-quality synthesis; insufficient citations
FixUse --timeout 60s for more retrieval time; try usearch deep for higher quality; verify ANTHROPIC_API_KEY is set
VerifyCitations appear in output with non-zero confidence score

Korean query returns no Korean results

FieldValue
Symptomusearch query --locale ko "..." returns only English results
CauseNaver adapter disabled or credentials missing
SPECSPEC-CLI-002
FixSet NAVER_CLIENT_ID + NAVER_CLIENT_SECRET; run usearch sources enable naver
Verifyusearch sources list shows naver as enabled; usearch sources test naver --query "test" succeeds

MCP Integration

Claude for Desktop shows “usearch MCP server not found”

FieldValue
SymptomMCP server not appearing in Claude
Causeusearch-mcp binary not in PATH, or config.json syntax error
SPECSPEC-MCP-001
Fix1. Verify which usearch-mcp. 2. Validate config.json with jq . config.json. 3. Restart Claude for Desktop
VerifyReload Claude — usearch appears in MCP servers list

MCP authentication fails

FieldValue
SymptomAUTH_ERROR in MCP response
CauseToken expired or invalid
FixGenerate a new token: usearch admin token create --name "claude-desktop" --expires 90d
Verifyusearch login --status shows valid session

Database & Storage

Qdrant connection refused

FieldValue
Symptomconnection refused on port 6333
CauseQdrant container not started or unhealthy
Fixdocker compose restart qdrant; check docker compose logs qdrant
Verifycurl http://localhost:6333/health returns {"status":"ok"}

Meilisearch returns stale results

FieldValue
Symptomusearch history search misses recent queries
CauseMeilisearch indexing lag (usually under 1s, but can be longer under load)
FixWait 2–3 seconds after running queries; check docker compose logs meilisearch
Verifycurl http://localhost:7700/indexes/usearch/stats shows updated document count

Authentication

usearch login hangs at browser step

FieldValue
SymptomBrowser doesn’t open or OAuth callback fails
CauseIncorrect OAUTH_REDIRECT_URL or firewall blocking the callback port
SPECSPEC-AUTH-002
FixVerify OAUTH_REDIRECT_URL=http://localhost:8081/auth/callback matches the OAuth provider config
VerifyComplete a test login flow in development

Performance

usearch deep times out

FieldValue
SymptomDeep research exits before completing
CauseDefault 5-minute timeout exceeded; or LLM API rate limits
FixIncrease timeout: usearch deep --timeout 15m "..."; check LLM API quota
VerifyShorter query completes within timeout

Getting More Help

  • GitHub Discussions  — community support
  • GitHub Issues  — bug reports
  • Enable debug logs: LOG_LEVEL=debug usearch query "..." — captures verbose output

When filing a bug report, include the output of usearch --version, docker compose ps, and the debug log output.
