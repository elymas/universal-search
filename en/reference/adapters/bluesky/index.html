Bluesky Adapter

beta(85.0% 7d)verified 2026-05-31

The Bluesky adapter queries the Bluesky AppView public search API via the AT Protocol. No authentication required. Part of the social Go package (internal/adapters/social/).

Shared implementation: Bluesky and X share the internal/adapters/social/ Go package. Implementation details (URL extraction, parse rules, scoring) are shared. See X adapter for the X-specific documentation.

Overview

Bluesky is a decentralised social platform based on the AT Protocol (atproto). The Universal Search Bluesky adapter queries the public AppView search endpoint (app.bsky.feed.searchPosts) with anonymous access. Results are sorted by top (relevance).

Use case: social post discovery, tech community content, real-time discussion search.

Setup

Authentication: not required — public endpoint.

The Bluesky AppView search API is publicly accessible without credentials. The adapter uses the SSRF guard allowlist: {public.api.bsky.app, api.bsky.app, bsky.app}.

# Verify the adapter is reachable:
usearch query --source bluesky "federated learning privacy"

Query syntax

Bluesky queries are forwarded to the AppView search API:

Universal Search fieldAT Protocol parameterNotes
Query.Textq={text}Full-text search across posts
Query.Langlang={bcp47}Language filter (e.g. ko, en)
Query.Sincesince={rfc3339}RFC 3339 timestamp filter
Query.MaxResultslimit={n}Capped at DefaultMaxResults

Sort: sort=top is hardcoded.

Rate limits

  • Advertised limit: 600 requests/min
  • Enforcement mechanism: HTTP 429 response handling — Bluesky AppView returns 429 with Retry-After
  • Upstream quota documentation: Bluesky API docs 
  • Exhaustion behaviour: Returns CategoryRateLimited with RetryAfter to the fanout dispatcher.
