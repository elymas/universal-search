## Overview

X (formerly Twitter) is a social platform. In Universal Search v1.0.0, the X adapter is a **compile-time stub** with no live search path wired. The `xCapabilities()` function returns `RateLimitPerMin: 0` and `DefaultMaxResults: 0` with the Notes field explicitly stating: _"DISABLED in v0. Set USEARCH_X_ENABLED=true to enable. SPEC-ADP-006-XENABLE pending. social sub-source stub; no live path wired."_

**The adapter will always return zero results in V1.** This is intentional and expected behaviour.

## Setup

Authentication: not required — adapter is disabled and makes no API calls.

There is no setup that will make this adapter functional in V1. Enabling `USEARCH_X_ENABLED=true` registers the adapter in the registry but the search path returns empty results because no HTTP client or API endpoint is wired.

```bash
# Do NOT use this adapter for production queries in V1:
# usearch query --source x "anything"  # returns 0 results
```

## Error reference

See [Error Taxonomy](/en/reference/adapters/errors) for the shared `Category` reference.

The X adapter in V1 returns empty results rather than errors. There are no X-specific error codes to document for V1.

When the adapter is invoked, `Search()` in `social/search_x.go` returns an empty slice without making any HTTP calls.
