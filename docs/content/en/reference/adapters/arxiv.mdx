## Overview

arXiv (arxiv.org) is a free distribution service and open-access archive for scholarly articles in physics, mathematics, computer science, quantitative biology, quantitative finance, statistics, electrical engineering, systems science, and economics. The Universal Search arXiv adapter queries the public XML/Atom feed API at `https://export.arxiv.org/api/query`.

**Use case:** academic paper discovery, research survey automation, citation-aware search.

## Setup

Authentication: not required â€” public endpoint.

The arXiv API is publicly accessible without credentials. No account registration or API key is needed. The adapter enforces a **3-second courtesy interval** between requests (configurable via `Options.MinRequestInterval`) as required by arXiv's API usage guidelines.

```bash
# Verify the adapter is reachable:
usearch query --source arxiv "transformer attention mechanism"
```

**Rate limit enforcement mechanism:** In-process interval guard â€” the adapter enforces a configurable minimum interval between consecutive requests in the same process. This is not an HTTP-429-based mechanism; arXiv does not return 429 responses. The `RateLimitPerMin: 20` value corresponds to the default 3-second interval.

## Query syntax

arXiv queries are submitted as free-text search strings. The adapter appends `sortBy=relevance` and maps Universal Search `Query` fields as follows:

| Universal Search field | arXiv query parameter | Notes |
|------------------------|----------------------|-------|
| `Query.Text` | `search_query=all:{text}` | Full-text search across title, abstract, author |
| `Query.Filters["category"]` | `search_query=cat:{category}` | e.g. `cs.AI`, `math.GT`, `quant-ph` |
| `Query.MaxResults` | `max_results={n}` | Capped at `DefaultMaxResults` when zero |
| `Query.Since` | Forward-compat placeholder | Date-range translation deferred to P2 |

**Score:** All results receive `Score: 0.5` (arXiv API does not expose per-paper relevance scores).

**LaTeX pass-through:** Mathematical notation in titles and abstracts is preserved as plain text.

## Rate limits

- **Advertised limit:** 20 requests/min (3-second interval guard)
- **Enforcement mechanism:** In-process interval guard (`search.go:142-146`) â€” no external rate limit enforced by arXiv at the API level
- **Upstream quota documentation:** [arXiv API access policy](https://info.arxiv.org/help/api/user-manual.html)
- **Exhaustion behaviour:** When concurrent callers share a process, the effective rate is lower than the per-caller advertised rate. The fanout dispatcher ([SPEC-FAN-001](/en/reference/architecture)) treats any non-429 error as `CategoryTransient` for retry purposes; the interval guard is an adapter-internal mechanism.

## Troubleshooting

| Symptom | Likely cause | Diagnostic command | Resolution | Related SPECs |
|---------|-------------|-------------------|------------|---------------|
| Zero results for broad query | arXiv category filter too narrow | Remove `category` filter, retry | Use full-text search without category constraint | SPEC-ADP-003 |
| Slow responses under load | Multiple concurrent callers share the interval guard | Check adapter logs for "rate guard sleep" messages | Deploy separate usearch instances per consumer to avoid interval contention | SPEC-FAN-001 |
| `CategoryUnavailable` on all arXiv queries | arXiv API outage | `curl https://export.arxiv.org/api/query?search_query=test&max_results=1` | Wait for arXiv recovery; see https://status.arxiv.org | SPEC-CACHE-001 |
