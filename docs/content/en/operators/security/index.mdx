## Current Security Posture (V1)

Until SPEC-SEC-001 merges, apply these baseline practices:

### Authentication
- Use OAuth/OIDC for all production deployments
- Rotate MCP tokens every 90 days
- Never use the default/example credentials in production

### Network
- Do not expose Qdrant (6333), Meilisearch (7700), or PostgreSQL (5432) ports to the public internet
- Use a reverse proxy (Caddy, nginx) with TLS for the public endpoint
- Restrict SearXNG to internal access only

### Secrets
- Generate strong secrets: `openssl rand -hex 32`
- Store secrets in a secrets manager (Vault, AWS SSM, Kubernetes Secrets)
- Never commit `.env` to version control

### Updates
- Monitor this repository for security advisories
- Keep Docker base images updated (run `docker compose pull` regularly)
- Subscribe to upstream security notices for SearXNG, Qdrant, Meilisearch

## Reporting Vulnerabilities

To report a security vulnerability in Universal Search, please open a **private** security advisory on the GitHub repository (Security tab → Report a vulnerability).

Do **not** report security vulnerabilities in public issues.
