## What This Runbook Will Cover

When SPEC-SEC-001 merges, this runbook will include:

- **Incident Detection** — how to identify a security incident in Universal Search
- **Severity Classification** — P0/P1/P2/P3 criteria for security incidents
- **Escalation Path** — who to notify and in what order
- **Containment Procedures** — immediate steps to limit blast radius
- **Evidence Collection** — log retention and forensic artifact preservation
- **Recovery Procedures** — restoring services after a security incident
- **Post-Mortem Template** — structured review format for lessons learned

## Immediate Contact

Until SPEC-SEC-001 merges, report security incidents by opening a **private** security advisory on the GitHub repository (Security tab → Report a vulnerability).

Do **not** report security incidents in public issues.
