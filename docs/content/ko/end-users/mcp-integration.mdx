# MCP 통합

Universal Search는 Claude for Desktop 및 기타 MCP 호환 클라이언트와 통합되는 MCP(Model Context Protocol) 서버를 제공합니다.

## 사전 요구사항

- Universal Search 스택 실행 중(`make compose-up`)
- Claude for Desktop 설치
- MCP 토큰(`usearch admin token create`로 발급)

## 설정

<Steps>

### MCP 토큰 발급

```bash
# 관리자로 로그인
usearch login --url http://localhost:8081

# Claude Desktop용 토큰 생성
usearch admin token create --name "claude-desktop" --expires 90d
# 출력: eyJhbGciOiJIUzI1NiJ9... (복사)
```

### Claude for Desktop 구성

`~/.config/claude-desktop/config.json` 파일 열기:
- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "usearch": {
      "command": "usearch-mcp",
      "args": ["--transport", "stdio"],
      "env": {
        "USEARCH_SERVER_URL": "http://localhost:8081",
        "USEARCH_MCP_TOKEN": "eyJhbGciOiJIUzI1NiJ9..."
      }
    }
  }
}
```

### Claude for Desktop 재시작

앱을 재시작하면 연결된 MCP 서버 목록에 "usearch"가 표시됩니다.

### 테스트

Claude for Desktop에서:
```
usearch MCP 서버를 사용해서 Raft 합의 알고리즘에 대한 정보를 찾아줘.
```

</Steps>

## 사용 가능한 도구

| 도구 | 설명 |
|---|---|
| `search` | 구성된 어댑터에서 검색 |
| `get_citation` | 인용 ID로 전체 문서 검색 |
| `list_sources` | 사용 가능한 검색 어댑터 목록 |
| `deep_research` | 멀티 에이전트 심층 연구 |

전체 문서는 [MCP 도구 레퍼런스](/en/reference/mcp)를 참조하세요 *(영어)*.

## 문제 해결

| 문제 | 해결 방법 |
|---|---|
| "서버를 찾을 수 없음" | `usearch-mcp` 바이너리가 PATH에 있는지 확인 |
| "인증 실패" | `USEARCH_MCP_TOKEN`이 유효하고 만료되지 않았는지 확인 |
| "연결 거부" | usearch 서버가 구성된 포트에서 실행 중인지 확인 |
